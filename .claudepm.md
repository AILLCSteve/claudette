# Claude PM Platform — Project Context

## Agent Instructions

When working on this project:
1. Read this file first to orient yourself
2. Check open obstacles in Claude PM before starting new work
3. Use `POST /api/sessions` to log completed work
4. Use `POST /api/obstacles` to escalate blockers that need human input
5. Update task statuses via `PATCH /api/tasks/[id]` as you work

## Stack Notes

- **Next.js 14** App Router with TypeScript — route groups `(app)` and `(auth)`, all server components unless `'use client'`
- **Supabase** for auth + Postgres — use `lib/supabase/server.ts` in server components/API routes, `lib/supabase/client.ts` in client components
- **Anthropic SDK** is server-side only — never import `@anthropic-ai/sdk` in client components
- **shadcn/ui** with Tailwind v3 HSL CSS variables — component stubs in `components/ui/`
- Pages that use Supabase must have `export const dynamic = 'force-dynamic'`
- Client components that create a Supabase client must do so lazily (inside event handlers), not at render time

## Key API Routes

- `POST /api/claude/chat` — AI portfolio intelligence (Anthropic, prompt-cached)
- `POST /api/claude/bootstrap` — generate agent session bootstrap prompt
- `POST /api/claude/generate-plan` — AI dev plan generation
- `POST /api/projects/import-marker` — import/sync from `.claudepm.md`
- `GET/POST /api/projects`, `GET/PATCH/DELETE /api/projects/[id]`
- `GET/POST /api/tasks`, `GET/PATCH/DELETE /api/tasks/[id]`
- `GET/POST /api/bugs`, `GET/PATCH/DELETE /api/bugs/[id]`
- `GET/POST /api/obstacles`, `GET/PATCH/DELETE /api/obstacles/[id]`
- `GET/POST /api/sessions`
- `GET/POST /api/agents`, `GET/PUT /api/agents/queue`

## Notes

- DB migrations are in `supabase/migrations/` — run in Supabase SQL Editor in order
- `.env.local` holds Supabase + Anthropic keys — never commit
- `render.yaml` configures Render deployment; `vercel.json` configures Vercel
- The `/pm-init` Claude command lives at `docs/claude-commands/pm-init.md`
